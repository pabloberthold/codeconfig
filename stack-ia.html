01 / REQUISITOS

Que tener listo antes de empezar

Vas a necesitar una máquina Linux con Podman instalado y, si querés aceleración por GPU, los drivers de NVIDIA + nvidia-container-toolkit.

# Podman
sudo dnf install podman podman-docker -y   # Fedora
sudo apt install podman -y                 # Ubuntu/Debian

# Drivers NVIDIA + container toolkit (opcional, para GPU)
sudo dnf install akmod-nvidia xorg-x11-drv-nvidia-cuda   # Fedora
# Para Ubuntu: instalá los drivers desde "Software & Updates → Additional Drivers"
# o con: sudo ubuntu-drivers autoinstall
sudo apt install nvidia-container-toolkit                 # ambos

# Verificar Podman
podman --version
podman info | grep -i host
02 / RED COMPARTIDA

Crear la red Podman

Para que los contenedores se vean entre sí por nombre, creamos una red tipo bridge:

podman network create stack-ia
03 / OLLAMA

Servidor de modelos locales

Ollama expone su API en el puerto 11434. Lo levantamos con la variable OLLAMA_HOST=0.0.0.0 para que sea accesible desde cualquier equipo de la red.

podman run -d \
  --name ollama \
  --network stack-ia \
  -p 11434:11434 \
  -v ollama_data:/root/.ollama \
  -e OLLAMA_HOST=0.0.0.0 \
  --hooks-dir=/usr/share/containers/oci/hooks.d/ \
  docker.io/ollama/ollama

Descargá un modelo liviano para empezar:

podman exec ollama ollama pull llama3.2:3b

Probá que responde:

curl http://localhost:11434/api/tags
GPU NVIDIA

El flag --hooks-dir es solo necesario si tenés GPU con nvidia-container-toolkit. Sin GPU, eliminá esa línea del comando.

04 / SEARXNG

Buscador autogestionado

SearXNG es un metabuscador que agrega resultados de Google, DuckDuckGo, Bing y otros sin rastreo. Open WebUI puede usarlo como motor de búsqueda para RAG.

Primero creá la carpeta de configuración:

mkdir -p ~/searxng && cd ~/searxng

Inicializá los archivos de configuración:

podman run -d --name searxng-tmp docker.io/searxng/searxng
podman cp searxng-tmp:/etc/searxng/settings.yml .
podman cp searxng-tmp:/etc/searxng/uwsgi.ini .
podman rm -f searxng-tmp
chmod 644 ~/searxng/settings.yml ~/searxng/uwsgi.ini

Ajustá settings.yml para permitir acceso desde la red local. Buscá estas líneas y cambialas:

# settings.yml — reemplazá estos valores
server:
  bind_address: "0.0.0.0"
  secret_key: "reemplaza-con-una-clave-segura"

Si querés desactivar el límite de requests (recomendado para LAN):

# settings.yml — agregá al final o donde esté la sección limiter
limiter:
  enabled: false
podman run -d \
  --name searxng \
  --network stack-ia \
  -p 4000:8080 \
  -v ~/searxng/settings.yml:/etc/searxng/settings.yml:ro,z \
  -v ~/searxng/uwsgi.ini:/etc/searxng/uwsgi.ini:ro,z \
  docker.io/searxng/searxng
05 / OPEN WEBUI

Interfaz web para Ollama

Open WebUI es el frontend conectado a Ollama. Se comunica con SearXNG a través del nombre del contenedor en la red compartida.

podman run -d \
  --name open-webui \
  --network stack-ia \
  -p 3000:8080 \
  -v open-webui_data:/app/backend/data \
  -e OLLAMA_BASE_URL=http://ollama:11434 \
  -e WEBUI_SECRET_KEY="cambia-esto-por-una-clave-aleatoria" \
  --restart unless-stopped \
  ghcr.io/open-webui/open-webui:main
06 / VERIFICACIÓN

Probar que todo funciona

En la misma máquina o desde cualquier dispositivo de la red local:

  • Ollama API — http://IP_DEL_HOST:11434/api/tags
  • SearXNG — http://IP_DEL_HOST:4000
  • Open WebUI — http://IP_DEL_HOST:3000

Si necesitás saber la IP de tu servidor:

ip addr show | grep "inet " | grep -v 127.0.0.1

En Open WebUI andá a Configuración → Conexiones y verificá que aparece Ollama conectado. Luego en Configuración → Web Search elegí SearXNG como motor de búsqueda, poné http://searxng:8080 como URL.

07 / SERVICIOS PERSISTENTES

Que arranquen solos al prender el equipo

Generá una unidad systemd por cada contenedor. Usamos --new para que systemd cree el contenedor desde cero al arrancar (no depende de que exista previamente).

# Habilitar linger para el usuario (permite servicios al iniciar sesión)
sudo loginctl enable-linger $USER

# Crear carpeta de unidades systemd
mkdir -p ~/.config/systemd/user

# Generar unidad para Ollama
podman generate systemd --name ollama --new > ~/.config/systemd/user/container-ollama.service

# Generar unidad para SearXNG
podman generate systemd --name searxng --new > ~/.config/systemd/user/container-searxng.service

# Generar unidad para Open WebUI
podman generate systemd --name open-webui --new > ~/.config/systemd/user/container-open-webui.service

# Recargar y habilitar todo
systemctl --user daemon-reload
systemctl --user enable container-ollama container-searxng container-open-webui

# Arrancar ahora (sin reiniciar)
systemctl --user start container-ollama container-searxng container-open-webui

# Ver estado
systemctl --user list-units 'container-*'

Opcional: creá un target que agrupe los tres para enable/start de un solo comando:

cat > ~/.config/systemd/user/stack-ia.target << 'EOF'
[Unit]
Description=Stack IA local — Ollama + SearXNG + Open WebUI
BindsTo=container-ollama.service container-searxng.service container-open-webui.service
After=container-ollama.service container-searxng.service container-open-webui.service

[Install]
WantedBy=default.target
EOF

systemctl --user daemon-reload
systemctl --user enable stack-ia.target
systemctl --user start stack-ia.target
08 / TIPS Y AJUSTES

Para el día a día

  • Más modelos: podman exec ollama ollama pull llama3.2:3b, mistral, phi4:14b. Corré varios al mismo tiempo.
  • GPU NVIDIA: si instalaste nvidia-container-toolkit, Ollama detecta la GPU automáticamente. Verificá con podman logs ollama | grep -i cuda o podman exec ollama ollama ps tras cargar un modelo.
  • Firewall: abrí los puertos necesarios. Con firewalld (Fedora/RHEL): sudo firewall-cmd --add-port=11434/tcp --add-port=3000/tcp --add-port=4000/tcp --permanent && sudo firewall-cmd --reload. Con ufw (Ubuntu/Debian): sudo ufw allow 11434/tcp && sudo ufw allow 3000/tcp && sudo ufw allow 4000/tcp.
  • Sin GPU: todo corre por CPU. Elegí modelos chicos como llama3.2:3b o phi4-mini:3.8b.
  • Elegí el nombre del host: ponele un DNS local como ia.local así no tenés que acordarte la IP.
  • Puertos personalizados: cambiá el lado izquierdo del -p si tenés conflictos (ej. -p 8080:8080 para Open WebUI).
