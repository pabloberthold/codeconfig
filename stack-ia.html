01 / REQUISITOS

Que tener listo antes de empezar

Vas a necesitar una máquina Linux con Podman instalado y, si querés aceleración por GPU, los drivers de NVIDIA + nvidia-container-toolkit.

# Podman
sudo dnf install podman podman-docker -y   # Fedora
sudo apt install podman podman-docker -y   # Ubuntu/Debian

# Drivers NVIDIA + container toolkit (opcional, para GPU)
sudo dnf install akmod-nvidia xorg-x11-drv-nvidia-cuda   # Fedora
sudo apt install nvidia-driver-xxx nvidia-utils-xxx       # Ubuntu
sudo apt install nvidia-container-toolkit                 # ambos

# Verificar Podman
podman --version
podman info | grep -i host
02 / RED COMPARTIDA

Crear la red Podman

Para que los contenedores se vean entre sí por nombre, creamos una red tipo bridge:

podman network create stack-ia
03 / OLLAMA

Servidor de modelos locales

Ollama expone su API en el puerto 11434. Lo levantamos con la variable OLLAMA_HOST=0.0.0.0 para que sea accesible desde cualquier equipo de la red.

podman run -d \
  --name ollama \
  --network stack-ia \
  -p 11434:11434 \
  -v ollama_data:/root/.ollama \
  -e OLLAMA_HOST=0.0.0.0 \
  --hooks-dir=/usr/share/containers/oci/hooks.d/ \
  docker.io/ollama/ollama

Descargá un modelo liviano para empezar:

podman exec ollama ollama pull llama3.2:3b

Probá que responde:

curl http://localhost:11434/api/tags
04 / SEARXNG

Buscador autogestionado

SearXNG es un metabuscador que agrega resultados de Google, DuckDuckGo, Bing y otros sin rastreo. Open WebUI puede usarlo como motor de búsqueda para RAG.

Primero creá la carpeta de configuración:

mkdir -p ~/searxng && cd ~/searxng

Inicializá los archivos de configuración:

podman run -d --name searxng-tmp docker.io/searxng/searxng
podman cp searxng-tmp:/etc/searxng/settings.yml .
podman cp searxng-tmp:/etc/searxng/uwsgi.ini .
podman rm -f searxng-tmp

Ajustá settings.yml para permitir la red local:

# settings.yml — buscá estas líneas y ajustalas
server:
  bind_address: "0.0.0.0"
  secret_key: "cambia-esto-por-una-clave-aleatoria"
  limiter: false          # desactivá si usás solo en LAN
podman run -d \
  --name searxng \
  --network stack-ia \
  -p 4000:8080 \
  -v ~/searxng/settings.yml:/etc/searxng/settings.yml:ro \
  -v ~/searxng/uwsgi.ini:/etc/searxng/uwsgi.ini:ro \
  -e SEARXNG_BASE_URL=http://0.0.0.0:4000/ \
  docker.io/searxng/searxng
05 / OPEN WEBUI

Interfaz web para Ollama

Open WebUI es el frontend conectado a Ollama. Se comunica con SearXNG a través del nombre del contenedor en la red compartida.

podman run -d \
  --name open-webui \
  --network stack-ia \
  -p 3000:8080 \
  -v open-webui_data:/app/backend/data \
  -e OLLAMA_BASE_URL=http://ollama:11434 \
  --restart unless-stopped \
  docker.io/ghcr.io/open-webui/open-webui:main
06 / VERIFICACIÓN

Probar que todo funciona

En la misma máquina o desde cualquier dispositivo de la red local:

  • Ollama API — http://IP_DEL_HOST:11434/api/tags
  • SearXNG — http://IP_DEL_HOST:4000
  • Open WebUI — http://IP_DEL_HOST:3000

Si necesitás saber la IP de tu servidor:

ip addr show | grep "inet " | grep -v 127.0.0.1

En Open WebUI andá a Configuración → Conexiones y verificá que aparece Ollama conectado. Luego en Configuración → Web Search elegí SearXNG como motor de búsqueda, poné http://searxng:8080 como URL.

07 / SERVICIOS PERSISTENTES

Que arranquen solos al prender el equipo

Con Podman + systemd podés generar unidades para cada contenedor usando podman generate systemd o simplemente con --restart always (que en Podman funciona con systemd user).

# Habilitar linger para el usuario (permite servicios al iniciar sesión)
sudo loginctl enable-linger $USER

# Generar unidad systemd para Open WebUI
mkdir -p ~/.config/systemd/user
podman generate systemd --name open-webui --files --new
systemctl --user enable container-open-webui
systemctl --user start container-open-webui

# Ver estado de todos los servicios
systemctl --user list-units 'container-*'

Si querés un solo comando para arrancar todo el stack al iniciar sesión, repetí el generate systemd con los otros contenedores.

08 / TIPS Y AJUSTES

Para el día a día

  • Más modelos: podman exec ollama ollama pull llama3.2:3b, mistral, phi4:14b. Corré varios al mismo tiempo.
  • GPU NVIDIA: si instalaste nvidia-container-toolkit y configuraste los hooks, Ollama detecta la GPU automáticamente. Verificá con podman exec ollama nvidia-smi.
  • Firewall: si usás firewalld, abrí los puertos: sudo firewall-cmd --add-port=11434/tcp --add-port=3000/tcp --add-port=4000/tcp --permanent && sudo firewall-cmd --reload.
  • Sin GPU: todo corre por CPU. Elegí modelos chicos como llama3.2:3b o phi4-mini:3.8b.
  • Elegí el nombre del host: ponele un DNS local como ia.local así no tenés que acordarte la IP.
  • Puertos personalizados: cambiá el lado izquierdo del -p si tenés conflictos (ej. -p 8080:8080 para Open WebUI).
