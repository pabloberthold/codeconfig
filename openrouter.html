01 / CUENTA

Creá tu cuenta y prepará el acceso

1

Entrá a OpenRouter e iniciá sesión o creá una cuenta.

2

Revisá el catálogo de modelos y la facturación. Podés usar modelos con distintos precios y límites desde la misma cuenta.

3

Si el modelo elegido no es gratuito, agregá créditos antes de ejecutar tareas largas.

02 / API KEY

Generá una clave para tu equipo

En Settings → Keys, seleccioná crear una nueva clave. Asignale un nombre reconocible —por ejemplo, laptop-codex— y un límite de gasto si aplica.

Copiala sólo una vez

Tratala como una contraseña. No la agregues a opencode.json, config.toml compartidos ni al repositorio.

03 / ENTORNO

Guardala fuera del código

Exportá la clave en la sesión actual. Para persistirla, usá un gestor de secretos o el mecanismo de credenciales de tu sistema operativo.

export OPENROUTER_API_KEY="sk-or-v1-..."

Después, configurá tu agente para leer OPENROUTER_API_KEY. Las guías de Codex y OpenCode de este sitio incluyen ejemplos concretos.

04 / VALIDACIÓN

Probá la clave antes de conectar un agente

Consultá la información de la clave sin imprimirla. Una respuesta válida confirma que la variable y la autenticación están bien configuradas.

curl https://openrouter.ai/api/v1/key \
  -H "Authorization: Bearer $OPENROUTER_API_KEY"

Para enviar solicitudes, OpenRouter usa https://openrouter.ai/api/v1 y modelos con el formato organización/modelo.

05 / SEGURIDAD

Rotá y limitá tus claves

  • Usá una clave distinta por equipo, proyecto o entorno.
  • Aplicá límites de gasto cuando el flujo lo permita.
  • Revocá de inmediato una clave expuesta y reemplazala en tus secretos.
