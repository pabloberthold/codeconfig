01 / INSTALACIÓN

Instalá la CLI

Elegí uno de estos métodos y verificá que el binario quedó disponible.

# Instalador standalone (macOS / Linux)
curl -fsSL https://chatgpt.com/codex/install.sh | sh

# npm (multiplataforma)
npm install -g @openai/codex

# Homebrew (macOS)
brew install --cask codex

codex --version

En Windows usá PowerShell: irm https://chatgpt.com/codex/install.ps1 | iex. Actualizás con el mismo comando de instalación.

02 / PRIMER INICIO

Autenticate y trabajá en un proyecto

1

2

3

cd mi-proyecto
codex
Configuración personal y por proyecto

Tus preferencias viven en ~/.codex/config.toml. Para compartir ajustes no sensibles con un repositorio, usá .codex/config.toml dentro del proyecto confiable.

03 / PROVEEDOR REMOTO

Usar OpenRouter

OpenRouter unifica 300+ modelos tras una API clave. Codex requiere el protocolo Responses API (el viejo wire_api = "chat" fue deprecado y ya no funciona).

export OPENROUTER_API_KEY="sk-or-..."

Agregá este bloque a ~/.codex/config.toml y reemplazá el modelo por un slug vigente del catálogo de OpenRouter.

model = "openai/gpt-5.3-codex"
model_provider = "openrouter"
model_reasoning_effort = "high"

[model_providers.openrouter]
name = "OpenRouter"
base_url = "https://openrouter.ai/api/v1"

[model_providers.openrouter.auth]
command = "sh"
args = ["-c", "echo $OPENROUTER_API_KEY"]
Ojo con el bloque auth

Usar env_key funciona para autenticar, pero Codex no puede traer el catálogo de modelos de OpenRouter. Preferí el bloque auth de arriba para que los slugs aparezcan correctamente.

04 / MODELOS EN TU EQUIPO

Ejecutar con Ollama

Codex ofrece modo OSS para modelos locales. Con Ollama instalado, descargá un modelo y ejecutá con --oss.

ollama pull gpt-oss:20b
codex --oss --local-provider ollama

Para usar otro modelo, pasalo con -m:

codex --oss --local-provider ollama -m gpt-oss:120b

Para fijar Ollama como proveedor OSS por defecto, agregá esto a ~/.codex/config.toml:

oss_provider = "ollama"
Consejo de rendimiento

Codex necesita al menos 64k de contexto. Elegí un modelo que entre en tu RAM y que soporte tool calling. gpt-oss:20b funciona en 16 GB; gpt-oss:120b requiere 80 GB.
