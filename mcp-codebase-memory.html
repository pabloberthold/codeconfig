01 / ¿QUÉ ES?

Un asistente estructural para tu agente

codebase-memory-mcp es un servidor MCP (Model Context Protocol) escrito en C puro que construye un grafo de conocimiento persistente de tu código base. Indexa funciones, clases, llamadas, rutas HTTP y conexiones entre servicios usando tree-sitter en 158 lenguajes, con resolución de tipos híbrida para Python, TypeScript, Go, Rust, Java, Kotlin, C++, C#, PHP y más.

No incluye un LLM — tu agente (Claude Code, Codex, OpenCode) es la capa de inteligencia que traduce lenguaje natural a consultas sobre el grafo.

99% menos tokens

Cinco consultas estructurales consumen ~3.400 tokens contra ~412.000 explorando archivo por archivo. Una consulta al grafo reemplaza docenas de ciclos de grep/read.

02 / INSTALACIÓN

Instalá el servidor MCP

El instalador oficial detecta tu sistema operativo, descarga el binario y configura los agentes que tengas instalados (Claude Code, Codex, OpenCode, VS Code, Cursor, Windsurf y más).

macOS / Linux (one-liner):

curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.sh | bash

Con visualizador de grafos 3D incluido:

curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.sh | bash -s -- --ui

Windows (PowerShell):

Invoke-WebRequest -Uri https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.ps1 -OutFile install.ps1
Unblock-File .\install.ps1
.\install.ps1

Opciones: --ui (visualizador), --skip-config (solo binario), --dir=ruta (ubicación personalizada).

Otros métodos:

  • npm: npm install -g codebase-memory-mcp
  • Homebrew: brew install codebase-memory-mcp
  • Arch Linux: yay -S codebase-memory-mcp-bin
  • go install: go install github.com/DeusData/codebase-memory-mcp@latest
03 / PRIMER USO

Indexá un proyecto

Una vez instalado, reiniciá tu agente y ejecutá este comando dentro de la sesión:

Indexá este proyecto

El agente traduce la instrucción a la herramienta index_repository del MCP. El indexado es inmediato para proyectos promedio (Django en ~6 segundos, Linux kernel en 3 minutos).

Para activar el indexado automático al conectar un proyecto nuevo:

codebase-memory-mcp config set auto_index true
04 / CLAUDE CODE

Usar con Claude Code

El instalador detecta Claude Code automáticamente y configura la entrada en ~/.claude.json. También instala un skill y tres agentes de grafo (Scout, Verify, Auditor) con hooks SessionStart y SubagentStart.

Para configuración manual, agregá esto a ~/.claude.json:

{
  "mcpServers": {
    "codebase-memory-mcp": {
      "command": "/ruta/completa/codebase-memory-mcp",
      "args": []
    }
  }
}

Verificá que el servidor esté activo con /mcp — deberías ver 15 herramientas disponibles.

05 / CODEX CLI

Usar con Codex CLI

El instalador configura Codex automáticamente. Agrega AGENTS.md, un skill y tres agentes de solo lectura con hooks SessionStart y SubagentStart.

Para configuración manual, agregá el servidor MCP a tu ~/.codex/config.toml:

[mcp_servers.codebase-memory-mcp]
command = "/ruta/completa/codebase-memory-mcp"
06 / OPENCODE

Usar con OpenCode

El instalador detecta OpenCode y configura el servidor en el archivo global de configuración. También agrega AGENTS.md, un skill y tres agentes de solo lectura con permisos denegados por defecto.

Para configuración manual, agregá esta entrada a tu ~/.config/opencode/opencode.json:

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "codebase-memory-mcp": {
      "type": "command",
      "command": "/ruta/completa/codebase-memory-mcp",
      "args": []
    }
  }
}
07 / LÍNEA DE COMANDOS

Comandos útiles

Todas las herramientas MCP también se pueden invocar desde la terminal:

# Indexar un proyecto
codebase-memory-mcp cli index_repository '{"repo_path": "/ruta/al/proyecto"}'

# Listar proyectos indexados
codebase-memory-mcp cli list_projects

# Buscar funciones por nombre
codebase-memory-mcp cli search_graph '{"project": "mi-proyecto", "name_pattern": ".*Handler.*"}'

# Trazar llamadas
codebase-memory-mcp cli trace_path '{"project": "mi-proyecto", "function_name": "buscar", "direction": "both"}'

# Consulta tipo Cypher
codebase-memory-mcp cli query_graph '{"project": "mi-proyecto", "query": "MATCH (f:Function) RETURN f.name LIMIT 5"}'

# Ver configuración
codebase-memory-mcp config list

# Auto-index al conectar
codebase-memory-mcp config set auto_index true

# Actualizar
codebase-memory-mcp update
08 / HERRAMIENTAS MCP

Las 15 herramientas del servidor

Indexado:

  • index_repository — Indexa un repositorio en el grafo
  • list_projects — Lista proyectos indexados con conteo de nodos/edges
  • delete_project — Elimina un proyecto del grafo
  • index_status — Estado del indexado de un proyecto

Búsqueda y consulta:

  • search_graph — Búsqueda estructurada por etiqueta, nombre, archivo
  • trace_path — Recorrido BFS: quién llama a una función y a quién llama
  • query_graph — Consultas tipo Cypher (solo lectura)
  • get_graph_schema — Esquema del grafo: tipos de nodos, edges, propiedades
  • get_code_snippet — Lee el código fuente de una función por nombre
  • search_code — Búsqueda de texto dentro de archivos indexados

Análisis:

  • get_architecture — Vista general: lenguajes, paquetes, rutas, clusters
  • detect_changes — Mapea cambios de git a símbolos afectados
  • manage_adr — CRUD de Architecture Decision Records
  • ingest_traces — Importa trazas de ejecución para validar conexiones HTTP
