Ollama + Podman

Ejecutá Ollama en un contenedor rootless, conservá los modelos en un volumen persistente y exponé la API sólo a tu propia máquina.

RUNTIMEPodman rootless
API LOCAL127.0.0.1:11434
EN ESTA GUÍA01 — Podman02 — Contenedor03 — Modelo04 — Prueba05 — Agentes
01 / REQUISITO

Instalá Podman

Podman permite ejecutar contenedores sin daemon y, normalmente, sin privilegios de root. Instalalo con el gestor de paquetes de tu distribución.

# Ubuntu / Debian
sudo apt update && sudo apt install -y podman

# Fedora
sudo dnf install -y podman

podman --version
02 / SERVICIO LOCAL

Iniciá Ollama con almacenamiento persistente

El volumen conserva los modelos aunque recrees el contenedor. El mapeo a 127.0.0.1 evita que la API quede publicada en toda tu red.

podman volume create ollama

podman run -d --name ollama --replace \
  -p 127.0.0.1:11434:11434 \
  -v ollama:/root/.ollama:Z \
  ollama/ollama

En sistemas sin SELinux, el sufijo :Z no suele ser necesario; en Fedora y derivados ayuda a etiquetar el volumen para el contenedor.

03 / MODELO

Descargá un modelo de código

Elegí un modelo acorde a tu memoria disponible y a las capacidades que necesitás. Para un agente, verificá soporte de herramientas y una ventana de contexto suficiente.

podman exec -it ollama ollama pull qwen3-coder
podman exec -it ollama ollama list
04 / VALIDACIÓN

Comprobá la API local

curl http://127.0.0.1:11434/api/tags

Si no responde, revisá podman ps y podman logs ollama. No publiques el puerto en 0.0.0.0 salvo que protejas la red y el acceso explícitamente.

05 / CONECTÁ TUS HERRAMIENTAS

Usalo con agentes de código

  • OpenCode: ollama launch opencode configura la integración oficial.
  • Codex: iniciá codex --oss --local-provider ollama.
  • API compatible: varios clientes usan http://localhost:11434/v1.
